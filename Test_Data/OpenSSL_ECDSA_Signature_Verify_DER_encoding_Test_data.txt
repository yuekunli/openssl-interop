

These are a set of test data for ECDSA digital signature.
This is to test the verification of the signature.
Signature is in the plain r|s concatenated format.
32 bytes 'r' and 32 bytes 's'.
"signatureTarget" is the message.
Remember what ECDSA receiver needs to verify the signature.
It needs the other party's public key and the plain text message and the r|s.
'g', 'p', 'q' are part of the public key. Public key also has a 'y' component.
OpenSSL's API assumes that the signature is in DER encoded format.
So when I have a plain r|s signature, I need to first DER encoded it and then
feed it to OpenSSL API. DER is more restrictive the BER in that it must not declare
a length longer than the value really needs.
If 'r' or 's' starts with 0 and the high bit of the next byte is also 0, which means
if I remove the first byte, the number still reads positive, then I should really remove the first byte.
